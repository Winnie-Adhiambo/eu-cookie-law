Why?

The EU passed a law some years ago that required all websites to warn about the dangers of cookies. I'm not going to comment on it here; if you want my opinion, please read this blogpost

It is now being enforced and some people are being fined, so it requires urgent action

In summary, Analytics cookies can't be set unless the user grants consent. Every country has a different interpretation of the EU directive, but in Spain, just displaying a "cookie banner" is not enough. No cookies can be set until the user takes action

This script helps webmasters track user action. Using the Spanish interpretation, consent is either (page 22 of the PDF)

  1. The user scrolls the webpage, while the notice is visible
  2. The user clicks on any link on the page

By using this script, webmasters can handle scrolling and clicks.

How does it work?

General idea:

  • All HTML files must include the cookieBanner div, which is hidden by default with CSS.
  • js/consent.js (which requires jQuery) will first check if there is a consent cookie.
  • If the cookie is present and grants consent, the banner will stay hidden and the script will execute the doConsent() function.
  • If the cookie is present and denies consent, the banner will stay hidden and no script will be executed
  • If there is no cookie, the banner will be displayed
